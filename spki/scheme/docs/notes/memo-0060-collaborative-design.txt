------------------------------------------------------------------------
ABSTRACT
------------------------------------------------------------------------

This memo documents the methodology of human-AI collaborative design as
practiced in the development of Cyberspace. The central insight: shared
context enables high-bandwidth communication. When both parties
understand the domain vocabulary, complex ideas compress to single
words, and conversation operates on deltas rather than explanations.

This is not a description of a tool. It is a case study in a new form of
collaboration, with Cyberspace as the artifact and the commit history as
evidence.


------------------------------------------------------------------------
1. THE BANDWIDTH PROBLEM
------------------------------------------------------------------------

Normal technical conversation has O(n) overhead. Every concept requires
explanation. Every term needs definition. Context must be re-established
with each interaction.

This creates a ceiling on complexity. Discussions stay shallow because
depth requires shared foundation that doesn't exist.

The alternative: build the foundation incrementally. Each conversation
adds to shared context. Each term defined becomes a compression key.
Over time, communication complexity shifts from O(n) to O(delta) - just
the changes.


------------------------------------------------------------------------
2. VOCABULARY AS COMPRESSION
------------------------------------------------------------------------

Cyberspace developed a domain vocabulary through iterative design:


  Term        Compressed Meaning                                                                           
  soup        Newton-style persistent object store, vault browser, content-addressed storage               
  weave       The distributed system as a whole, nodes connected, lambdas flowing                          
  forge       Build system, password generation, incremental compilation, JIT - all metallurgical shaping  
  smelter     Raw material refinement - word lists to digraph statistics, source to compiled modules       
  residents   Editors that extend top-level, blade guards disengaged, Murphy/Shrayer heritage              
  seal        Cryptographic commitment, hash chain, immutability guarantee                                 
  realm       Trust domain, capability scope, federated identity boundary                                  
  wilderness  The space between realms, untrusted, discovery zone                                          

Each term carries history. 'Forge' invokes Gasser's Multics work,
Piper/Callas at DEC, VMS 6.0. One word, forty years of context.


------------------------------------------------------------------------
3. THE ITERATIVE LOOP
------------------------------------------------------------------------

The collaboration follows a pattern:

  * Human provides vision and constraints - what should exist, what matters, what the heritage requires
  * AI provides implementation and exploration - code, alternatives, edge cases discovered
  * Human evaluates and refines - 'keep it', 'too ugly', 'add author credit'
  * AI incorporates and extends - the delta applied, implications traced
  * Context accumulates - next iteration starts from higher ground

Neither party could build this alone. The human has the vision and
heritage but not the bandwidth for implementation. The AI has the
bandwidth but not the taste or history. Together: vision realized at
scale.


------------------------------------------------------------------------
4. DELTA COMMUNICATION
------------------------------------------------------------------------

Examples from a single session (2026-01-28):


    User: "lambda completions broken at open paren"
    AI: [knows: lineage-src.c, findLastWord, paren_wrap mode, completion callback]
        [understands: user typed '(sta', TAB failed, need '(' as word boundary]
        [fixes: one line change to findLastWord, rebuild, test, commit]

No explanation needed. 'Lambda completions' identifies the subsystem.
'Open paren' identifies the boundary condition. The fix is obvious given
shared context.

Contrast with cold start:


    User: "When I type open paren then 'sta' and press TAB in the
           Scheme REPL using the linenoise-based line editor with
           command completion enabled in schemer mode where completions
           should be wrapped in parentheses, the completion doesn't
           trigger because the word-finding function only splits on
           spaces, not on open parentheses, so it sees '(sta' as the
           word to complete rather than 'sta'..."

Same information. 10x the words. That's the compression ratio of shared
context.


------------------------------------------------------------------------
5. CONTEXT AS FORCE MULTIPLIER
------------------------------------------------------------------------

Accumulated context changes what's possible:

  * First session: explain SPKI, explain vaults, explain why Scheme
  * Tenth session: 'add seek to command aliases' - done in seconds
  * Hundredth session: 'residents with prefixes, blade guards, heritage credits' - complete spec

The collaboration gets more powerful over time. Each session builds on
all previous sessions. The shared dictionary grows. Compression
improves. Bandwidth increases.

This is the opposite of typical tool use, where each session starts
fresh.


------------------------------------------------------------------------
6. EVIDENCE: THE COMMIT HISTORY
------------------------------------------------------------------------

The Cyberspace git history documents the collaboration:

  * Commits co-authored by human and AI
  * Feature branches showing iterative refinement
  * Commit messages that reference shared vocabulary
  * Code that reflects both heritage (VMS, SPKI, Scheme) and modern practice

The artifact is not just the system. It is the recorded process of
building the system.


------------------------------------------------------------------------
7. IMPLICATIONS FOR HUMAN-AI COLLABORATION
------------------------------------------------------------------------

Lessons from this case study:

  * Context persistence matters - collaboration without memory is hobbled
  * Vocabulary emergence is natural - don't force terminology, let it arise from use
  * The human provides taste - knowing what to keep, what to discard, what feels right
  * The AI provides throughput - implementing, exploring, handling mechanical transformation
  * Neither dominates - this is partnership, not tool use

The methodology scales. Any sufficiently complex domain could benefit
from this approach: accumulated context, shared vocabulary, delta
communication, iterative refinement.
